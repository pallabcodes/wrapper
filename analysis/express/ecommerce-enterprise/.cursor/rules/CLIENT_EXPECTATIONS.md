# Client Expectations

This project must feel native to enterprise internal teams.

## Tone and delivery
1. Non experimental tone and predictable behavior.
2. Changes are conservative and backward compatible by default.
3. Documentation is clear and directly actionable.

## DX and API shape
1. Public APIs are typed and minimal.
2. Defaults are sensible and secure.
3. Configuration is discoverable and explicit.

## Review standards
1. Code readability and maintainability prioritized.
2. Tests and examples demonstrate real usage.
3. Commit messages and PR descriptions are clear.
