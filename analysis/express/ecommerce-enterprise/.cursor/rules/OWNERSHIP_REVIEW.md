# Ownership and Review

## Ownership
1. Use CODEOWNERS to route reviews to maintainers and principals
2. Each package must list owners in CODEOWNERS
3. Ownership implies accountability for quality and releases

## Review triggers
1. Large diffs above a threshold require principal review
2. Breaking changes require design review before code
3. Security sensitive changes require security review

## PR expectations
1. Scope is focused and commits are descriptive
2. Checklists in template are completed
3. Linked issues and ADRs where applicable
